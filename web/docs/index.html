Dokumentacja phaselimiter phaselimiter documentation

phaselimiter to narzedzie do masteringu audio oparte na silniku bakuage/phase_limiter (fork filipszoldra). Przetwarza pliki audio uzywajac wielopasmowego mastera referencyjnego i fazowego limitera opartego na optymalizacji FFT.

phaselimiter is an audio mastering tool built on the bakuage/phase_limiter engine (filipszoldra fork). It processes audio using a reference-based multiband master and a phase limiter driven by FFT optimization.

Szybki start Quick start

1
Wgraj plik audio
Upload an audio file
Kliknij strzalke w gornym panelu lub przeciabnij plik na obszar aplikacji. Obslugiwane formaty: WAV, MP3, FLAC, AAC, M4A, OGG, AIFF.
Click the arrow in the top panel or drag a file onto the app. Supported formats: WAV, MP3, FLAC, AAC, M4A, OGG, AIFF.
2
Analiza (opcjonalna, ale zalecana)
Analysis (optional, but recommended)
Kliknij "Analyze" aby uzyskac profil spektralny, LUFS, LRA i sugestie ustawien. Silnik automatycznie zaproponuje korekcje EQ per-pasmo na podstawie porownania z profilem referencyjnym.
Click "Analyze" to get a spectral profile, LUFS, LRA, and setting suggestions. The engine automatically proposes per-band EQ corrections based on comparison with the reference profile.
3
Dostosuj ustawienia
Adjust settings
Ustawienia domyslne dzialaja dobrze dla wiekszosci nagrań. Jesli chcesz - dostosuj cel glosnosci (LUFS), intensywnosc masteringu i inne parametry.
Default settings work well for most tracks. Optionally adjust the loudness target (LUFS), mastering intensity, and other parameters.
4
Uruchom mastering
Run mastering
Kliknij "Master". Postep pojawi sie w kolejce. Czas przetwarzania: typowo 3-8 minut dla nagrania 3-5 min (zalezy od ustawien jakosci).
Click "Master". Progress appears in the job queue. Processing time: typically 3-8 minutes for a 3-5 min track (depends on quality settings).
5
Pobierz wynik i porownaj
Download result and compare
Po zakonczeniu kliknij "Download". Panel "Input / Output comparison" pojawi sie automatycznie z porownaniem metryk, wykresow EQ, spektrogramow i odtwarzaczem A/B.
When done, click "Download". The "Input / Output comparison" panel appears automatically with metric comparison, EQ charts, spectrograms, and an A/B player.
ⓘ Plik wyjsciowy jest przesylany do przegladarki w trakcie masteringu (przez to samo polaczenie SSE). Pobieranie nie wymaga osobnego polaczenia z serwerem i dziala nawet po zrestartowaniu instancji serwerowej. The output file is transferred to the browser during mastering (over the same SSE connection). Download does not require a separate server connection and works even after a server instance restart.

Opis kontrolek Controls reference

1
Cel glosnosci (Loudness Target)
Loudness Target
Glosnosc-20 .. 0 LUFS
Loudness-20 .. 0 LUFS
Docelowa glosnosc wyjsciowa w LUFS. Domyslnie: -14 LUFS (standard Spotify/YouTube). Wyzsza wartosc = glosniej = mocniejsze dzialanie limitera i wiecej przyduszenia dynamiki. Zalecane wartosci: -14 (streaming), -9 do -12 (radio/telewizja), -9 (agresywny mix).
Target output loudness in LUFS. Default: -14 LUFS (Spotify/YouTube standard). Higher value = louder = harder limiting and more dynamic compression. Recommended: -14 (streaming), -9 to -12 (radio/TV), -9 (aggressive mix).
Wskazowka: zacznij od -14 LUFS. Jesli brzmienie jest zbyt skompresowane - obniz do -16 lub -18.
Tip: start at -14 LUFS. If the result sounds over-compressed, lower to -16 or -18.
2
AutoMastering5
AutoMastering5
Intensywnosc0 .. 1
Intensity0 .. 1
Jak mocno silnik dopasowuje barwe i dynamike do wzorca referencyjnego. Domyslnie: 0.4. Niskie wartosci (0.1-0.3) = subtelna korekcja. Wysokie (0.6-0.8) = wyrazna zmiana tonalna.
How strongly the engine matches tone and dynamics to the reference. Default: 0.4. Low values (0.1-0.3) = subtle correction. High (0.6-0.8) = strong tonal change.
Dopasowanie stereo0 .. 1
Stereo match0 .. 1
Dopasowanie szerokosci obrazu stereo do wzorca referencyjnego. 0 = bez zmian w stereo, 1 = pelne dopasowanie. Domyslnie: 1.0.
Matches the stereo width to the reference profile. 0 = no stereo change, 1 = full match. Default: 1.0.
Zachowaj basy
Preserve bass
Chroni niskie czestotliwosci przed nadmiernym przyduszeniem przez AutoMastering5. Zalecane wlaczone (domyslnie).
Protects low frequencies from over-compression by AutoMastering5. Recommended: on (default).
Tryb EQ
EQ Mode
Ceiling + transform (domyslny) - pelne dzialanie: ogranicza gorny puchar optymalizatora per-pasmo i realizuje korekcje tonalna.
Ceiling only - tylko ograniczenie optimum, bez korekcji tonalnej. Mniej "pompowania" przy silnej korekcji.
Transform only - tylko statyczna korekcja tonalna (przesunięcie barwy bez ingerencji w optimum).
Ceiling + transform (default) - full effect: caps optimizer upper bound per band and applies tonal correction.
Ceiling only - only limits the optimizer ceiling, no tonal correction. Less pumping with strong corrections.
Transform only - only static tonal correction (color shift without touching the optimizer).
Korekcja EQ per-pasmo
Per-band EQ correction
Reczna korekcja tonalna - przeciagnij punkty krzywej EQ w gore lub w dol. Wartosc 0 dB = neutralna. Zaznacz "Enable EQ correction" aby aktywowac. Dostepne po kliknieciu "Analyze" (sugestie sa wypelniane automatycznie).
Manual tonal correction - drag EQ curve points up or down. 0 dB = neutral. Check "Enable EQ correction" to activate. Available after clicking "Analyze" (suggestions are filled in automatically).
Tylko limiter
Limiter Only
Tryb diagnostyczny: omija AutoMastering5 i stosuje tylko phase limiter + normalizacje glosnosci. Przydatne do porownania "przed/po" AutoMastering5.
Diagnostic mode: bypasses AutoMastering5 and applies only the phase limiter + loudness normalization. Useful for comparing with/without AutoMastering5.
3
Pre-kompresja
Pre-compression
Wlaczony/wylaczony
Enable/disable
Lagodna kompresja stosowana przed phase limiterem. Zmniejsza pompowanie i trzaski. Domyslnie: wlaczona.
Gentle compression applied before the phase limiter. Reduces pumping and clicks. Default: on.
Prog (offset)0 .. +18 dB
Threshold offset0 .. +18 dB
Offset progu kompresji powyzej sredniej glosnosci wejscia. Domyslnie: +6 dB. Nizszy = kompresja wchodzi wczesniej i jest mocniejsza. Wyzszy = tylko szczyty sa kompresowane.
Threshold offset above the input track's average loudness. Default: +6 dB. Lower = compression kicks in earlier and harder. Higher = only peaks are compressed.
Okno czasowe0.05 .. 1 s
Time window0.05 .. 1 s
Czas usredniania do obliczenia aktualnej glosnosci. Domyslnie: 0.2 s. Krotkie wartosci = szybka reakcja, wiecej artefaktow. Dlugie = plynna kompresja, mniej pompowania.
Averaging window for current loudness calculation. Default: 0.2 s. Short = fast response, more artifacts. Long = smooth compression, less pumping.
4
Phase Limiter
Nadprobkowanie1x / 2x / 4x
Oversampling1x / 2x / 4x
Wyzsze nadprobkowanie zmniejsza aliasing i artefakty intermodulacji. 1x (domyslne) = najszybsze, najmniejsze zuzycie RAM. 4x = najczysciej, ale wymaga ~4x wiecej CPU i RAM.
Higher oversampling reduces aliasing and intermodulation artifacts. 1x (default) = fastest, least RAM. 4x = cleanest, but requires ~4x more CPU and RAM.
Jakosc (iteracje)40 .. 400
Quality (iterations)40 .. 400
Liczba iteracji optymalizatora FFT. Domyslnie: 100. Wiecej iteracji = mniej artefaktow limitera, ale wolniejsze przetwarzanie. Wartosc 40-60 = szybki podglad. 200-400 = maksymalna jakosc, duzo wolniej.
Number of FFT optimizer iterations. Default: 100. More iterations = fewer limiting artifacts, but slower processing. 40-60 = quick preview. 200-400 = maximum quality, much slower.
5
Sufit (Ceiling)
Ceiling
True-peak sufit-3 .. 0 dBTP
True-peak ceiling-3 .. 0 dBTP
Maksymalny true-peak wyjscia. Domyslnie: -1.0 dBTP. Standard bezpieczny dla platform streamingowych (Spotify, YouTube, Apple Music). -1 dBTP zapobiega przekroczeniu 0 dBFS po kompresji MP3/AAC.
Maximum true-peak of the output. Default: -1.0 dBTP. The safe standard for streaming platforms (Spotify, YouTube, Apple Music). -1 dBTP prevents 0 dBFS clipping after MP3/AAC encoding.
Wskazowka: pozostaw -1.0 dBTP dla streamingu. Dla masteringu na CD mozna ustawic -0.1 dBTP.
Tip: keep -1.0 dBTP for streaming. For CD mastering you can use -0.1 dBTP.
6
Mastering sekcyjny
Section-aware mastering
Sekcje wlaczone
Sections enabled
Pozwala oznaczac fragmenty nagrania (np. ciche intro, spokojny bridge) gdzie mastering ma dzialac lagodniej lub mocniej niz w reszcie utworu. Aktywne sekcje sa mieszane z sygnalem suchym zgodnie z intensywnoscia sekcji.
Lets you mark segments of the recording (e.g. quiet intro, calm bridge) where mastering should be applied more gently or more strongly than the rest of the track. Active sections are blended with the dry signal according to section intensity.
Intensywnosc sekcji0 .. 1
Section intensity0 .. 1
Udzial przetworzonego sygnalu w zaznaczonych sekcjach. 0 = sekcja pozostaje zupelnie sucha (bez masteringu), 1 = pelny mastering. Domyslnie: 0.75.
Wet signal share within marked sections. 0 = section stays completely dry (no mastering), 1 = full mastering. Default: 0.75.

Jak to dziala - pipeline How it works - pipeline

Ponizej schemat przetwarzania sygnalu wewnatrz silnika phase_limiter. Kazdy etap jest wykonywany seryjnie na pelnym pliku audio.

Below is the signal processing pipeline inside the phase_limiter engine. Each stage processes the full audio file sequentially.

Wejscie (dekodowanie)
Input (decoding)
ffmpeg dekoduje plik wejsciowy do PCM. Obslugiwane formaty: WAV, MP3, FLAC, AAC, M4A, OGG, AIFF i inne.
ffmpeg decodes the input file to PCM. Supported formats: WAV, MP3, FLAC, AAC, M4A, OGG, AIFF and others.
AutoMastering5
Wielopasmowy korektor i kompresor mid/side. Optymalizacja rozniczkowa (differential evolution) dobiera parametry aby dopasowac brzmienie do profilu referencyjnego (mastering_reference.json). Kontroluje: intensywnosc, dopasowanie stereo, EQ per-pasmo.
Multiband equalizer and mid/side compressor. Differential evolution optimization selects parameters to match the sound to the reference profile (mastering_reference.json). Controls: intensity, stereo match, per-band EQ.
Korekcja EQ (analityczna)
Analytic EQ correction
Statyczna korekcja per-pasmo wyliczona z analizy wejscia i porownania z profilem referencyjnym. Maksymalna korekcja: +/-6 dB, skalowana przez intensywnosc masteringu.
Static per-band correction calculated from input analysis compared to the reference profile. Maximum correction: +/-6 dB, scaled by mastering intensity.
Pre-kompresja
Pre-compression
Opcjonalna lagodna kompresja z progiem powyzej sredniej glosnosci wejscia. Zmniejsza szczytowe wartosci przed fazowym limiterem, co redukuje artefakty limitowania.
Optional gentle compression with a threshold above the input's average loudness. Reduces peak levels before the phase limiter, which reduces limiting artifacts.
Normalizacja do celu LUFS
Normalization to target LUFS
Liniowy gain dopasowuje glosnosc sygnalu do docelowego LUFS przed limiterem.
Linear gain adjusts the signal loudness to the target LUFS before the limiter.
Phase Limiter (GradCalculator)
Phase Limiter (GradCalculator)
Glowny modul. Iteracyjna optymalizacja FFT minimalizuje naruszenia progu true-peak bez typowego "twardego przycięcia" (clipping). Kazda iteracja przesuwa fazy skladowych widma aby zmniejszyc szczyty przy zachowaniu brzmienia. Liczba iteracji: kontrolka Quality (40-400).
Main module. Iterative FFT optimization minimizes true-peak threshold violations without hard clipping. Each iteration shifts the phases of spectral components to reduce peaks while preserving sound quality. Iteration count: Quality control (40-400).
Sufit true-peak
True-peak ceiling
Ostateczne przycięcie true-peak do ustawionego sufitu (domyslnie -1.0 dBTP). Gwarantuje, ze plik wyjsciowy nie przekroczy limitu nawet po ewentualnym kodowaniu stratnym.
Final true-peak clamp to the configured ceiling (default: -1.0 dBTP). Guarantees the output does not exceed the limit even after lossy encoding.
Wyjscie (WAV)
Output (WAV)
Plik WAV 44.1 kHz, 16-bit lub 24-bit. Przesylany do przegladarki przez SSE i dostepny od razu po zakonczeniu masteringu.
WAV file at 44.1 kHz, 16-bit or 24-bit. Transferred to the browser via SSE and available immediately after mastering finishes.

Slowniczek terminow Glossary

LUFS
Loudness Units Full Scale - miara glosnosci postrzeganej (percepcyjnej), mierzona zgodnie z ITU-R BS.1770. -14 LUFS = standard Spotify i YouTube. Im wyzsza wartosc (blizej 0), tym glosniej.
Loudness Units Full Scale - a perceptual loudness measure per ITU-R BS.1770. -14 LUFS = Spotify and YouTube standard. Higher values (closer to 0) = louder.
LRA
Loudness Range - rozstep dynamiczny miedzy cichymi a glosnymi fragmentami, w jednostkach LU. Mala LRA (np. 3-5 LU) = mocno skompresowany, plaska dynamika. Duza LRA (np. 12+ LU) = naturalna dynamika, zmienna glosnosc.
Loudness Range - dynamic spread between quiet and loud sections, in LU. Small LRA (e.g. 3-5 LU) = heavily compressed, flat dynamics. Large LRA (e.g. 12+ LU) = natural dynamics, varying loudness.
True Peak
Szczytowa wartosc sygnalu po interpolacji analogowej (rekonstrukcji). Rozni sie od "sample peak" - mierzy rzeczywisty analog po konwersji D/A. Wazne dla platform streamingowych: MP3/AAC moga wzmacniac szczyty, dlatego mastering celuje w -1 dBTP, a nie 0 dBFS.
Peak signal level after analog interpolation (reconstruction). Different from "sample peak" - measures the actual analog level after D/A conversion. Important for streaming: MP3/AAC encoding can boost peaks, so mastering targets -1 dBTP rather than 0 dBFS.
Dynamics
Odchylenie standardowe krotkoterminowej glosnosci w dB. Wyzsze wartosci = bardziej dynamiczny dzwiek z wyraznymi zmianami glosnosci. Niskie wartosci = brzmienie mocno skompresowane, "glosne".
Standard deviation of short-term loudness in dB. Higher values = more dynamic sound with noticeable loudness changes. Low values = heavily compressed, "loud" sound.
AutoMastering5
Glowny modul masteringowy silnika. Wielopasmowy korektor i kompresor mid/side, ktorego parametry sa dobierane automatycznie przez optymalizacje rozniczkowa (differential evolution) aby dopasowac brzmienie do profilu referencyjnego.
Main mastering module of the engine. A multiband equalizer and mid/side compressor whose parameters are automatically selected by differential evolution optimization to match the reference sound profile.
Phase Limiter
Limiter oparty na iteracyjnej optymalizacji FFT. Zamiast "obcinac" szczyty (clipping), przesuwa fazy skladowych widmowych aby zredukowac amplitudy szczytowe przy minimalnym wplywie na brzmienie.
A limiter based on iterative FFT optimization. Instead of hard-clipping peaks, it shifts the phases of spectral components to reduce peak amplitudes with minimal impact on sound quality.
ERB
Equivalent Rectangular Bandwidth - psychoakustyczna skala pasm czestotliwosci odpowiadajaca percepcji ucha ludzkiego. Silnik przetwarza 9 pasm ERB zamiast liniowej skali Hz.
Equivalent Rectangular Bandwidth - a psychoacoustic frequency band scale corresponding to human hearing perception. The engine processes 9 ERB bands rather than a linear Hz scale.
Pre-compression
Etap lagodnej kompresji przed glownym limiterem. Redukuje naglee szczyty (transjenty) co pozwala limiterowi pracowac plynniej i z mniej slyszalnymi artefaktami ("pompowanie").
A gentle compression stage before the main limiter. Reduces sudden peaks (transients), allowing the limiter to work more smoothly with fewer audible artifacts ("pumping").
EQ
Equalization - korekcja tonalna przez wzmacnianie lub tłumienie pasm czestotliwosci. W kontekscie tej aplikacji: korekcja per-pasmo ERB wynikajaca z analizy lub ustawiana recznie.
Equalization - tonal correction by boosting or cutting frequency bands. In this app: per-ERB-band correction derived from analysis or set manually.
Spectrogram
Wizualizacja tresci czestotliwosciowej w czasie: os X = czas (od lewej do prawej), os Y = czestotliwosc (nizej = basowe, wyzej = wysokie), jasnosc = energia. Widoczny w panelu analizy i porownania.
A visualization of frequency content over time: X axis = time (left to right), Y axis = frequency (bottom = bass, top = highs), brightness = energy. Visible in the analysis and comparison panel.
dBTP
decibel True Peak - jednostka pomiaru true-peak. -1 dBTP oznacza ze true-peak jest 1 dB ponizej pelnej skali (0 dBFS). Rozni sie od dBFS (sample peak) dla nagrań po kodowaniu stratnym.
decibel True Peak - unit for measuring true-peak level. -1 dBTP means the true-peak is 1 dB below full scale (0 dBFS). Differs from dBFS (sample peak) for recordings after lossy encoding.
SSE
Server-Sent Events - protokol HTTP do jednokierunkowego strumieniowania danych z serwera do przegladarki. Uzywany do przesylania postepu masteringu i pliku wyjsciowego w czasie rzeczywistym.
Server-Sent Events - an HTTP protocol for one-way server-to-browser data streaming. Used to transmit mastering progress and the output file in real time.
