phaselimiter-gui — controls reference

Guide to mastering settings and their effect on the audio. The Advanced section targets glitches — crunch, pumping, clicks, smearing — that persist even at gentle loudness and intensity.

Quick summary. Hearing crunch at -13 LUFS / intensity 0.4? That's not loudness — it's the limiter, ceiling and pre-compression stages. Fix them in the Advanced section.

Basic controls Basic

Output directory

Folder where mastered files are saved (defaults to Downloads / Desktop). Output files get the suffix _output.wav.

Target loudness → --reference

Range: −20 … 0 LUFS  ·  Default: −9 LUFS

The biggest lever. Sets how loud the track is pushed before the limiter. Lower (e.g. −13 / −14) = less gain into the limiter = less crunch and pumping. Recommended −12 to −14 for music with preserved dynamics.

Mastering intensity → --mastering5_mastering_level

Range: 0.0 … 1.0  ·  Default: 1.0

How strongly AutoMastering5 reshapes the tone toward the learned reference (per-band M/S compressor, tuned by differential-evolution optimisation). Lower = gentler, closer to the original.

Note. In mastering5 mode (the only mode used here) only this flag has any effect. --mastering_matching_level and --mastering_ms_matching_level apply only to classic mode and are not passed.

Preserve bass → --erb_eval_func_weighting

Enables perceptual weighting in the limiter to protect low frequencies from over-limiting.

Advanced — glitch reduction Advanced

These stages run independently of loudness and intensity — they can degrade the sound even at gentle basic settings.

Limiter only (diagnostic) → --mastering false

Default: off

Skips AutoMastering5, runs only the limiter. Use for A/B diagnosis: if quiet sections stop distorting, the global reference-matching is the culprit — lower Mastering intensity (e.g. to 0.2–0.3). Not a production mode.

True-peak ceiling (dB) → --ceiling

Range: −3.0 … 0.0 dB  ·  Default: −1.0 dB  ·  Target: clicks

Maximum inter-sample peak. At 0 dB, peaks exceed 0 dBFS after MP3/AAC encoding. −1.0 dB provides headroom and eliminates most clicks.

Limiter oversampling (×) → --limiter_internal_oversample

Options: 1 / 2 / 4  ·  Default: 1  ·  Target: crunch / aliasing

Hard limiting at 44.1 kHz creates aliasing. Running at 2× or 4× removes most of it. Recommended 2×. CPU/RAM scales with the factor.

Limiter quality (iterations) → --max_iter1

Range: 100 … 400 (step 50)  ·  Default: 100  ·  Target: distortion smearing

The limiter is an iterative FISTA optimizer. Too few iterations = audible error (crunch) and pre-ringing (transient smearing). Try 200 if you hear distortion.

Pre-compression → --pre_compression

Default: on  ·  Target: pumping

Loudness compressor running before the limiter. Can be the source of pumping. Uncheck to preserve full dynamics.

Pre-comp threshold (dB) → --pre_compression_threshold

Range: 0 … 18 dB  ·  Default: 6 dB

How many dB above measured loudness the pre-compressor activates. Higher = less pumping.

Pre-comp window (s) → --pre_compression_mean_sec

Range: 0.05 … 1.0 s  ·  Default: 0.2 s

Averaging window. Extending to ~0.4 s reduces pumping.

Analysis & suggestions New

Analyze a track & suggest settings

Opens a file picker, measures the chosen audio file (audio_analyzer), and auto-fills every control with gentle values derived from LRA, true-peak and spectral balance. Non-WAV inputs are decoded via ffmpeg automatically.

After analysis the Detected quiet sections list is also populated from the short-term loudness curve.

Before/after report (double-click a finished row)

Opens a dialog with: a metrics scorecard (LUFS, LRA, true-peak, dynamics, stereo, sound quality — input vs. output with warnings), analysis images (spectrogram, spectrum balance, stereo image), and the list of detected quiet sections.

Detected quiet sections Advanced

The algorithm finds spans where short-term loudness sits ≥9 LU below the track's loud-section reference (95th-percentile of the loudness curve). A global master over-processes these — AutoMastering5 is fitted to whole-song statistics (dominated by loud sections) and boosts near-empty bands during sparse material.

Editing the list

  • + Add — manually add a section by entering start and end in seconds.
  • Remove — delete the selected row.

Section intensity

Range: 0.0 … 1.0  ·  Default: 0.25

The gentler mastering level used to re-render quiet sections.

Section-aware mastering (checkbox)

When enabled and sections are present:

  1. The engine masters the whole track normally (preserves the loudness arc).
  2. Each quiet section is re-mastered from the original input with the gentler level.
  3. The rescued section is loudness-matched to the global master at that position.
  4. It is spliced back with 80 ms cross-fades at the boundaries.
If any rescue step fails, the output falls back to the global master — the base render is never lost.

Reference tone (EQ) Advanced

Collapsed expander in the main window.

AutoMastering5 targets a reference — a per-ERB-band array of mid_mean values (target mono level in dB): the AI's target tonal curve. The four sliders tilt it without recompiling the engine:

SliderFrequency rangeEffect at +dB
Low<250 HzBassier target
Low-mid250–2000 HzWarmer mids
High-mid2000–6000 HzMore presence / brightness
High>6000 HzMore air

Range: ±6 dB (clamped in code)  ·  Default: 0 (no change)

The GUI writes a modified copy of mastering_reference.json to a temp file and passes it via --mastering5_mastering_reference_file. The covariance, side_mean and all other fields are preserved verbatim.

⚠ Over-boosting an empty band (e.g. +6 dB High on a bright-free track) fights the limiter and can worsen artifacts. Start with ±2 dB and verify with the before/after spectrum image.
