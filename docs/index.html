Files — input & output

Input files

Drag audio files onto the drop zone, or click + Add files. Supported: WAV, FLAC, MP3, AIFF, and any format ffmpeg can decode. Multiple files are processed as a queue — each gets its own mastering job.

Output folder

Folder where mastered files are saved. Defaults to your Downloads folder. Click Browse to pick a different location. The output folder is shared across all jobs in the current session.

Output filename

Only used when a single file is in the queue. For multiple files, each output uses the original filename with _output.wav appended.

1

Loudness target

the biggest lever

Target loudness --reference

Range: −20 … 0 LUFS  ·  Default: −14 LUFS  ·  Recommended: −9 to −14 LUFS

Sets how loud the engine pushes the track before the phase limiter. This is the single biggest control over the final sound. Lower values (e.g. −14) mean the limiter receives less gain, which directly reduces crunch, pumping, and true-peak clicking.

  • −9 LUFS — streaming loudness norm; little headroom, strong limiting.
  • −14 LUFS — YouTube/Spotify norm; comfortable dynamics, less crunch.
  • −18 or quieter — mastering for later chain / archives.
If you hear crunch or pumping: lower this first (try −14 to −16) before touching any other control.
2

AutoMastering5

tone & stereo reshaping

AutoMastering5 (AM5) is the AI core of the engine. It is a per-band mid/side compressor whose parameters are found by differential-evolution optimisation to match a learned reference profile (mastering_reference.json). It adjusts tonal balance and stereo width toward the reference, independently of the loudness target.

Intensity --mastering5_mastering_level

Range: 0.0 … 1.0  ·  Default: 0.4  ·  Recommended: 0.3–0.5

How strongly AM5 reshapes your track toward the learned reference tone and dynamics.

  • 0 — AM5 has no effect (still runs, but output = input).
  • 0.4 — gentle reshaping; original character preserved.
  • 1.0 — maximum reference matching; strong coloration.

Intensity also scales the EQ correction below: your per-band modification is multiplied by intensity inside the engine (capped at ±6 dB).

Stereo match --mastering_ms_matching_level

Range: 0.0 … 1.0  ·  Default: 1.0

How strongly AM5 reshapes the stereo field (mid/side balance per band) toward the reference. At 1.0, stereo width per band is fully matched. Lowering to 0.5–0.7 preserves your original stereo image and reduces over-widening of hi-hats or spatial smearing on sparse material.

Analysis EQ correction --eq_analysis_target

When you add a file, the engine analyzes its per-band spectrum and draws a red curve (your track's current level per band). Enable EQ correction to activate the editor:

  • Pink curve — your desired modification in dB (±12 dB per band). Drag it up or down to boost or cut each frequency band.
  • Dashed red curve + red fill — predicted applied result: modification × intensity, capped at ±6 dB. This is what the engine will actually apply. Move the Intensity slider and the prediction updates live.
  • The center frequency of each band is shown below its name.

The engine applies this static per-band gain after AM5 and before pre-compression. The GUI sends the raw modification array; intensity scaling and the ±6 dB clamp happen inside the engine.

Warning: boosting an empty band (e.g. +12 dB Air on a track with very little high content) fights the limiter and can worsen artifacts. Check the before/after comparison after each EQ change.

Preserve bass --erb_eval_func_weighting

Applies perceptual weighting in the phase limiter that protects low-frequency content from over-limiting. Keep this on for most music — especially bass-heavy genres.

Advanced: per-band optimizer --mastering5_eq_band_levels / --mastering5_eq_transform_levels

A 9-band curve (0–2, 1 = neutral) that restrains how aggressively AM5 reshapes each frequency band. Two modes:

  • Ceiling — scales the optimizer's per-band wet-gain upper bound (soft penalty in the cost function). Below 1: the optimizer is penalised for boosting that band. Above 1: it is allowed more room.
  • Transform — scales the realized post-optimization wet-gain (deterministic multiplicative). Below 1: the output is pulled back toward neutral regardless of what the optimizer chose.
  • Ceiling + Transform — applies both simultaneously.
  • Symmetric transform — if on, Transform also scales cuts (not just boosts). Without it, only boosts are scaled — cuts are unaffected.

The curve is only sent to the engine when at least one band differs from 1.0.

Diagnostic: Limiter only --mastering false

Skips AM5 entirely and runs only the phase limiter + ceiling. Use for A/B diagnosis: if the track sounds clean in Limiter-only mode but artifacts appear with AM5 on, the reference-matching is the culprit — lower Intensity. Not a production mode.

3

Pre-compression

tames pumping before the limiter

A loudness-domain compressor that runs before the phase limiter. It evens out the loudest moments so the limiter receives a more consistent level, reducing the per-hit crunch and pumping in the final output.

Enabled --pre_compression

Default: on. When off, the signal goes from AM5 straight into the limiter with full dynamics. Disable it to preserve transient impact, or when pumping is caused by this stage.

Threshold offset --pre_compression_threshold

Range: 0 … 18 dB  ·  Default: +6 dB above measured loudness

How many dB above the track's measured integrated loudness the compressor activates. Higher offset = it activates less often = more dynamics, less pumping.

Window --pre_compression_mean_sec

Range: 0.05 … 1.0 s  ·  Default: 0.2 s

Averaging window for the loudness measurement driving the compressor. A longer window (~0.4 s) gives a smoother, slower action that reduces pumping artifacts. A shorter window reacts faster and is more effective at catching sharp transients.

4

Phase limiter

the crunch stage — FFT optimizer

The phase limiter is an iterative FISTA optimizer working in the frequency domain. It finds the smallest-error modification to the spectrum that brings all samples below the ceiling. This is the primary source of crunch and transient smearing — more resources = fewer artifacts.

Oversampling --limiter_internal_oversample

Options: 1× / 2× / 4×  ·  Default: 1×

The limiter operates at the file's native sample rate (usually 44.1 kHz). Hard limiting at that rate creates inter-sample aliasing — a harsh metallic crunch. Oversampling temporarily raises the rate so aliasing artifacts fold back above audible range.

  • 1× — fastest; audible aliasing on transient-heavy material.
  • 2× — recommended for final masters; removes most aliasing.
  • 4× — cleanest; significant CPU and RAM increase.

Quality (iterations) --max_iter1

Range: 40 … 400  ·  Default: 100

Number of FISTA iterations. Too few = the optimizer does not converge fully → audible error (residual crunch) and pre-ringing (transient smearing before each loud peak). Try 200 for final masters. 400 is very slow but maximally accurate.

5

Ceiling

true-peak headroom

True-peak ceiling --ceiling

Range: −3.0 … 0.0 dB  ·  Default: −1.0 dB

The maximum allowed inter-sample true-peak level. Setting it to 0 dBFS means that after MP3 or AAC encoding, reconstructed samples can exceed 0 dBFS and clip in the decoder → audible clicks. −1.0 dB provides 1 dB of headroom and eliminates most encoder-induced true-peak clipping. For Spotify/YouTube compliance, −1.0 to −1.5 dB is standard.

6

Section-aware mastering

gentle blend for quiet parts

AM5 is optimised against the whole-song statistics — dominated by the loud sections. During quiet intros, breaks, and outros, it tries to match a "loud-section reference," often over-boosting empty frequency bands → distortion on sparse material. Section-aware mastering solves this by blending quiet sections toward the dry (unprocessed) signal in a single engine pass.

Blend quiet sections toward dry signal --mastering5_section_ranges

Master checkbox. When enabled and at least one section exists, the engine uses the section list and section intensity setting. The sections are detected automatically from the loudness timeline, or drawn manually.

Loudness timeline & sections

After analysis, a chart shows the track's short-term loudness over time. Shaded bands mark detected or manually drawn quiet sections. Click anywhere on the chart to place a playhead — then use ▶ Play from here to audition that part of the track from your default audio player.

Drawing sections manually

Click + Draw section to enter draw mode (cursor becomes a crosshair). Then click and drag on the timeline to define a new section. Release the mouse to commit it (minimum width: 1 second). To cancel without drawing, click ✕ Cancel.

Sections are also editable in the list below the chart — each entry shows start and end times with a Remove button.

Section intensity --mastering5_section_intensity

Range: 0.0 … 1.0  ·  Default: 0.75

Controls the wet/dry blend strength inside the marked sections:

  • 1.0 — full AM5 processing (same as normal, no blending).
  • 0.75 — 75% AM5 + 25% dry; gentle reduction.
  • 0.0 — fully dry (unprocessed) inside sections.

A 1-second raised-cosine ramp is applied at each section boundary to prevent clicks and audible transitions. The blend happens in a single engine pass — no re-rendering or splicing.

Typical starting point: 0.15–0.40 for heavily over-processed intros. For subtle reduction, try 0.60–0.75.

Analysis & comparison

Analyze button

Opens a file picker and runs full analysis on the selected track using audio_analyzer (non-WAV files are decoded via ffmpeg first). The analysis runs in a background goroutine so the UI stays fully responsive during the potentially long analysis (the progress bar animates at the top of the window). When finished:

  • The per-band spectrum chart in Card 2 is filled.
  • The controls are updated with glitch-avoiding suggested values (loudness, intensity, section ranges, section intensity, preserve bass).
  • The analysis drawer opens with full metrics.

Analysis drawer (Analysis ▾)

The side drawer shows metrics from the last analyzed file:

  • Integrated LUFS — average loudness of the whole track.
  • True-peak — highest inter-sample peak (dBTP).
  • LRA — Loudness Range; the dynamic spread between quiet and loud passages.
  • Dynamics — a peak-to-loudness measure of transient presence.
  • Sample rate and Duration.
  • Spectrogram — time-frequency energy map with Hz labels on the Y axis (20 kHz at top, 0 Hz at bottom) and time on the X axis. Click the ⤢ button or the image to open it full-screen in the lightbox.

Spectrogram lightbox

Click any spectrogram image (in the drawer or in the job comparison) to open it full-screen. The lightbox shows the image at native resolution with the frequency axis (Hz labels) on the left and the time axis at the bottom. Press Esc or click the backdrop to close.

Mastering comparison (job result)

Expand a finished job in the queue to see the before/after comparison:

  • Metrics — input vs output LUFS, true-peak, LRA with deltas.
  • Per-band EQ chart — the dB change AM5 applied in each of the 9 bands, with the delta value printed under each band name.
  • Loudness over time — short-term loudness of input and output overlaid.
  • Input / Output spectrograms — side by side, both zoomable.

Symptom → fix

cheat sheet
What you hearFirst fixThen
Clicks / crackles Ceiling → −1.0 dB Output as WAV; avoid MP3 at 0 dBFS
Crunch on peaks Oversampling → 2× Quality → 200; lower Target loudness (try −14)
Pumping / breathing Pre-comp window → 0.4 s Raise threshold offset; or disable pre-compression
Smearing / loss of clarity Quality → 200 Lower Intensity (try 0.3)
Wrong tone — too bright/dark Enable EQ correction → drag the band(s) Watch the dashed-red prediction; verify in comparison
Distortion on quiet intro / break Section-aware + section intensity 0.15–0.40 Draw sections manually if auto-detection missed them
Kick deformation / hi-hat over-widening Per-band optimizer: lower Sub/Low (kick) or High/V-hi/Air (hi-hats) Lower Stereo match to 0.5–0.7
Track sounds over-processed / generic Lower Intensity to 0.2–0.3 Lower Stereo match; use EQ correction instead of AM5 for tone
Change one parameter at a time and compare by ear — otherwise it is impossible to tell what actually helped. Use the before/after comparison panel.

Docelowa głośność

największa dźwignia

Target loudness --reference

Zakres: −20 … 0 LUFS  ·  Domyślnie: −14 LUFS  ·  Zalecane: −9 do −14 LUFS

Określa, jak głośno silnik wyrówna utwór przed phase limiterem. Jest to najważniejsza pojedyncza kontrolka. Niższe wartości (np. −14) oznaczają mniejsze wzmocnienie trafiające w limiter, co bezpośrednio redukuje chrupanie, pompowanie i trzaski true-peak.

  • −9 LUFS — norma głośności streamingowej; mało zapasu, mocne limitowanie.
  • −14 LUFS — norma YouTube/Spotify; komfortowa dynamika, mniej chrupania.
  • −18 lub ciszej — mastering do dalszego łańcucha / archiwum.
Słyszysz chrupanie lub pompowanie? Najpierw obniż tę wartość (spróbuj −14 do −16), zanim dotkniesz czegokolwiek innego.
2

AutoMastering5

kształtowanie tonu i stereo

AutoMastering5 (AM5) to rdzeń AI silnika. Jest to kompresja mid/side per-pasmo, której parametry wyznacza optymalizacja różnicowo-ewolucyjna dopasowując utwór do wyuczonego wzorca (mastering_reference.json). Dostosowuje balans tonalny i szerokość stereo względem wzorca, niezależnie od docelowej głośności.

Intensywność (Intensity) --mastering5_mastering_level

Zakres: 0,0 … 1,0  ·  Domyślnie: 0,4  ·  Zalecane: 0,3–0,5

Jak mocno AM5 przekształca utwór w stronę wyuczonego wzorca tonu i dynamiki.

  • 0 — AM5 nie ma efektu (działa, ale wyjście = wejście).
  • 0,4 — delikatne kształtowanie; oryginalny charakter zachowany.
  • 1,0 — maksymalne dopasowanie do wzorca; silna koloryzacja brzmienia.

Intensywność skaluje też korekcję EQ poniżej: Twoja modyfikacja per pasmo jest mnożona przez intensywność wewnątrz silnika (przycięta do ±6 dB).

Dopasowanie stereo (Stereo match) --mastering_ms_matching_level

Zakres: 0,0 … 1,0  ·  Domyślnie: 1,0

Jak mocno AM5 przekształca pole stereo (balans mid/side per pasmo) w stronę wzorca. Przy 1,0 szerokość stereo per pasmo jest w pełni dopasowywana. Obniżenie do 0,5–0,7 zachowuje więcej oryginalnego obrazu stereo i redukuje nadmierne poszerzanie hihatsów lub rozmywanie przestrzeni na rzadkim materiale.

Korekcja EQ (per pasmo) --eq_analysis_target

Po dodaniu pliku silnik analizuje widmo per pasmo i rysuje czerwoną krzywą (bieżący poziom Twojego utworu per pasmo). Włącz EQ correction, żeby aktywować edytor:

  • Różowa krzywa — Twoja pożądana modyfikacja w dB (±12 dB per pasmo). Przeciągnij ją w górę lub dół, żeby wzmocnić lub osłabić dane pasmo częstotliwości.
  • Czerwona przerywana krzywa + czerwone wypełnienie — przewidywany zastosowany efekt: modyfikacja × intensywność, przycięty do ±6 dB. To jest to, co silnik faktycznie zastosuje. Ruch suwakiem Intensywności aktualizuje podgląd na żywo.
  • Częstotliwość środkowa każdego pasma jest wyświetlana pod jego nazwą.

Silnik stosuje ten statyczny zysk per pasmo po AM5 i przed pre-kompresją. GUI wysyła surową tablicę modyfikacji; skalowanie przez intensywność i przycięcie ±6 dB dzieje się wewnątrz silnika.

Uwaga: wzmacnianie pustego pasma (np. +12 dB Air na utworze z bardzo małą ilością wysokich) walczy z limiterem i może pogorszyć artefakty. Sprawdź porównanie przed/po po każdej zmianie EQ.

Zachowaj bas (Preserve bass) --erb_eval_func_weighting

Stosuje percepcyjne ważenie w phase limiterze chroniące niskie częstotliwości przed nadmiernym ograniczaniem. Trzymaj włączone dla większości muzyki — szczególnie gatunków z dominującym basem.

Zaawansowane: optymalizator per pasmo --mastering5_eq_band_levels / --mastering5_eq_transform_levels

Krzywa 9-pasmowa (0–2, 1 = neutralnie) ograniczająca agresywność AM5 per pasmo. Dwa tryby:

  • Ceiling (sufit) — skaluje górną granicę wet-gain optymalizatora per pasmo (miękka kara w funkcji kosztu). Poniżej 1: optymalizator jest karany za wzmacnianie pasma. Powyżej 1: dostaje więcej swobody.
  • Transform — skaluje zrealizowany wet-gain po optymalizacji (deterministyczne mnożenie). Poniżej 1: wyjście jest cofane w stronę neutralności niezależnie od wyboru optymalizatora.
  • Ceiling + Transform — oba jednocześnie.
  • Symmetric transform — gdy włączony, Transform skaluje też cięcia (nie tylko wzmocnienia). Bez tego cięcia nie są skalowane.

Krzywa jest wysyłana do silnika tylko gdy co najmniej jedno pasmo różni się od 1,0.

Diagnostyka: tylko limiter --mastering false

Pomija AM5 całkowicie i uruchamia tylko phase limiter + sufit. Tryb diagnostyczny A/B: jeśli utwór brzmi czysto w trybie Limiter-only, ale artefakty pojawiają się z AM5 — dopasowanie do wzorca jest winowajcą → obniż Intensywność. Nie jest to tryb produkcyjny.

3

Pre-kompresja

tłumi pompowanie przed limiterem

Kompresor głośnościowy działający przed phase limiterem. Wyrównuje najgłośniejsze momenty, żeby limiter otrzymywał bardziej stabilny poziom, co redukuje pompowanie i chrupanie per uderzenie na wyjściu.

Włączona (Enabled) --pre_compression

Domyślnie: włączona. Gdy wyłączona, sygnał przechodzi z AM5 prosto do limitera z pełną dynamiką. Wyłącz ją by zachować uderzenie transjentów albo gdy pompowanie pochodzi z tego etapu.

Próg — offset (Threshold offset) --pre_compression_threshold

Zakres: 0 … 18 dB  ·  Domyślnie: +6 dB ponad zmierzoną głośność

O ile dB ponad zmierzoną zintegrowaną głośność utworu kompresor zaczyna działać. Wyższy offset = aktywuje się rzadziej = więcej dynamiki, mniej pompowania.

Okno (Window) --pre_compression_mean_sec

Zakres: 0,05 … 1,0 s  ·  Domyślnie: 0,2 s

Okno uśredniania głośności sterujące kompresorem. Dłuższe okno (~0,4 s) daje płynniejsze, wolniejsze działanie i redukuje pompowanie. Krótsze okno reaguje szybciej i jest skuteczniejsze przy ostrych transjentach.

4

Phase limiter

etap chrupania — optymalizator FFT

Phase limiter to iteracyjny optymalizator FISTA działający w domenie częstotliwości. Szuka modyfikacji widma o minimalnym błędzie, która sprowadza wszystkie próbki poniżej sufitu. To jest główne źródło chrupania i rozmywania transjentów — więcej zasobów = mniej artefaktów.

Nadpróbkowanie (Oversampling) --limiter_internal_oversample

Opcje: 1× / 2× / 4×  ·  Domyślnie: 1×

Limiter działa przy natywnej częstotliwości próbkowania pliku (zwykle 44,1 kHz). Twarde limitowanie przy tej częstotliwości tworzy aliasing — metaliczne chrupanie. Nadpróbkowanie tymczasowo podnosi częstotliwość, żeby artefakty aliasingowe zwinęły się powyżej słyszalnego zakresu.

  • 1× — najszybsze; słyszalny aliasing na materiale bogatym w transjenty.
  • 2× — zalecane na finalne mastery; usuwa większość aliasingów.
  • 4× — najczystsze; znaczny wzrost CPU i RAM.

Jakość — iteracje (Quality) --max_iter1

Zakres: 40 … 400  ·  Domyślnie: 100

Liczba iteracji FISTA. Za mało = optymalizator nie zbiega w pełni → słyszalny błąd (resztkowe chrupanie) i pre-ringing (rozmywanie transjentów przed każdym głośnym szczytem). Spróbuj 200 na finalne mastery. 400 jest bardzo wolne, ale maksymalnie dokładne.

5

Sufit

zapas true-peak

Sufit true-peak --ceiling

Zakres: −3,0 … 0,0 dB  ·  Domyślnie: −1,0 dB

Maksymalny dozwolony poziom true-peak między próbkami. Ustawienie 0 dBFS oznacza, że po kodowaniu MP3 lub AAC zrekonstruowane próbki mogą przekroczyć 0 dBFS i obciąć się w dekoderze → słyszalne trzaski. −1,0 dB daje 1 dB zapasu i eliminuje większość trzasków wywołanych przez koder. Dla Spotify/YouTube standard to −1,0 do −1,5 dB.

6

Mastering świadomy sekcji

delikatny blend dla cichych fragmentów

AM5 jest optymalizowane względem statystyk całego utworu — zdominowanych przez głośne sekcje. Podczas cichych intro, breaków i outro próbuje dopasować „wzorzec głośnych sekcji", często nadmiernie wzmacniając puste pasma częstotliwości → zniekształcenia na rzadkim materiale. Mastering świadomy sekcji rozwiązuje to, mieszając ciche fragmenty w stronę sygnału dry (nieprzetworzonego) w jednym przebiegu silnika.

Mieszaj ciche sekcje w stronę dry --mastering5_section_ranges

Główny checkbox. Gdy włączony i istnieje co najmniej jedna sekcja, silnik używa listy sekcji i ustawienia intensywności sekcji. Sekcje są wykrywane automatycznie z wykresu głośności lub rysowane ręcznie.

Wykres głośności i sekcje

Po analizie wykres pokazuje krótkoterminową głośność w czasie. Zacienione pasy oznaczają wykryte lub ręcznie narysowane ciche sekcje. Kliknij w dowolnym miejscu wykresu, żeby umieścić głowicę odtwarzania — a potem użyj ▶ Play from here, żeby odsłuchać ten fragment z domyślnego odtwarzacza.

Rysowanie sekcji ręcznie

Kliknij + Draw section, żeby wejść w tryb rysowania (kursor staje się celownikiem). Następnie kliknij i przeciągnij na wykresie, żeby zdefiniować nową sekcję. Zwolnij mysz, żeby zatwierdzić (minimalna szerokość: 1 sekunda). Żeby anulować bez rysowania, kliknij ✕ Cancel.

Sekcje można też edytować na liście pod wykresem — każdy wpis pokazuje czas startu i końca oraz przycisk Remove.

Intensywność sekcji (Section intensity) --mastering5_section_intensity

Zakres: 0,0 … 1,0  ·  Domyślnie: 0,75

Kontroluje siłę blendu wet/dry wewnątrz zaznaczonych sekcji:

  • 1,0 — pełne przetwarzanie AM5 (tak samo jak normalnie, bez mieszania).
  • 0,75 — 75% AM5 + 25% dry; delikatna redukcja.
  • 0,0 — całkowicie dry (nieprzetworzone) wewnątrz sekcji.

Rampa 1 sekundy (raised-cosine) jest stosowana na każdej granicy sekcji, żeby zapobiec kliknięciom i słyszalnym przejściom. Blend odbywa się w jednym przebiegu silnika — bez re-renderowania ani sklejania.

Typowy punkt startowy: 0,15–0,40 dla mocno przetworzonych intro. Przy subtelnej redukcji spróbuj 0,60–0,75.

Analiza i porównanie

Przycisk Analyze

Otwiera okno wyboru pliku i uruchamia pełną analizę wybranego utworu przy użyciu audio_analyzer (pliki inne niż WAV są wcześniej dekodowane przez ffmpeg). Analiza działa w tle (goroutine), więc UI pozostaje w pełni responsywne podczas potencjalnie długiej analizy (pasek postępu animuje się na górze okna). Po zakończeniu:

  • Wykres widma per pasmo w Karcie 2 zostaje wypełniony.
  • Kontrolki zostają zaktualizowane z sugerowanymi wartościami unikającymi glitchy (głośność, intensywność, zakresy sekcji, intensywność sekcji, zachowaj bas).
  • Panel boczny analizy otwiera się z pełnymi metrykami.

Panel analizy (Analysis ▾)

Panel boczny pokazuje metryki z ostatnio analizowanego pliku:

  • Zintegrowane LUFS — średnia głośność całego utworu.
  • True-peak — najwyższy szczyt między próbkami (dBTP).
  • LRA — Loudness Range; rozpiętość dynamiczna między cichymi i głośnymi fragmentami.
  • Dynamics — miara transjentów (szczyt do głośności).
  • Sample rate i Duration.
  • Spektrogram — mapa energii czas-częstotliwość z etykietami Hz na osi Y (20 kHz u góry, 0 Hz u dołu) i czasem na osi X. Kliknij przycisk ⤢ lub sam obraz, żeby otworzyć go na pełnym ekranie w lightboxie.

Lightbox spektrogramu

Kliknij dowolny obraz spektrogramu (w panelu lub w porównaniu zadania), żeby otworzyć go na pełnym ekranie. Lightbox pokazuje obraz w natywnej rozdzielczości z osią częstotliwości (etykiety Hz) po lewej stronie i osią czasu na dole. Naciśnij Esc lub kliknij tło, żeby zamknąć.

Porównanie masteringu (wynik zadania)

Rozwiń zakończone zadanie w kolejce, żeby zobaczyć porównanie przed/po:

  • Metryki — LUFS, true-peak, LRA wejściowe vs wyjściowe z deltami.
  • Wykres EQ per pasmo — zmiana dB zastosowana przez AM5 w każdym z 9 pasm, z wartością delta wypisaną pod nazwą każdego pasma.
  • Głośność w czasie — krótkoterminowa głośność wejścia i wyjścia nakładka.
  • Spektrogramy wejście/wyjście — obok siebie, oba z możliwością powiększenia.
