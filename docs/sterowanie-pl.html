phaselimiter-gui — opis kontrolek

Przewodnik po ustawieniach masteringu i ich wpływie na dźwięk. Sekcja Zaawansowane służy do usuwania „glitchy”, które zostają nawet przy spokojnym poziomie głośności i intensywności.

Skrót dla niecierpliwych. Jeśli słyszysz trzaski, chrupanie albo „pompowanie” mimo ustawienia np. -13 LUFS i intensywności 0,4 — to nie wina głośności. Winne są etapy limitera, sufitu (ceiling) i pre‑kompresji. Ich kontrolki znajdziesz w sekcji Zaawansowane poniżej.

Ustawienia podstawowe Podstawowe

Katalog wyjściowy (Output directory)

Folder, do którego zapisywane są zmasterowane pliki (domyślnie Pobrane / Pulpit). Plik wynikowy otrzymuje przyrostek _output.wav.

Docelowa głośność (Target loudness) → flaga --reference

Zakres: −20 … 0 LUFS  ·  Domyślnie: −9 LUFS

Najważniejszy „duży” regulator. Określa, jak głośno ma być wyrównany utwór przed limiterem. Niższa wartość (np. −13 / −14) = mniej wzmocnienia wpychanego w limiter = mniej chrupania i pompowania. Zalecane −12 do −14 dla muzyki z zachowaną dynamiką.

Intensywność masteringu (Mastering intensity)

Zakres: 0,0 … 1,0  ·  Domyślnie: 1,0

Steruje trzema flagami naraz: --mastering_matching_level, --mastering_ms_matching_level oraz --mastering5_mastering_level. Określa, jak mocno algorytm przekształca brzmienie w stronę wyuczonego wzorca (korekcja pasmowa + kompresja M/S). Niżej = delikatniej, bliżej oryginału.

Zachowaj bas (Preserve bass) → flaga --erb_eval_func_weighting

Włącza percepcyjne ważenie błędu limitera tak, aby chronić niskie częstotliwości przed nadmiernym ograniczaniem.

Uwaga. Jeżeli zjechałeś już z głośnością i intensywnością, a artefakty zostają — zostaw te suwaki w spokoju i przejdź do sekcji Zaawansowane. To tam są właściwe lekarstwa.

Zaawansowane — redukcja glitchy Zaawansowane

Te etapy działają niezależnie od głośności i intensywności — dlatego potrafią psuć dźwięk nawet przy spokojnych ustawieniach podstawowych.

Sufit true‑peak (dB) (True‑peak ceiling) → --ceiling

Zakres: −3,0 … 0,0 dB  ·  Domyślnie: −1,0 dB  ·  Cel: trzaski / przesterowania

Maksymalny poziom szczytu międzypróbkowego (true‑peak). Przy 0 dB limiter celuje dokładnie w 0 dBFS, a po konwersji/kodowaniu (zwłaszcza MP3/AAC) szczyty międzypróbkowe „wyskakują” ponad zero i słychać trzaski. Ustawienie −1,0 dB daje zapas i eliminuje większość kliknięć.

Nadpróbkowanie limitera (×) (Limiter oversampling) → --limiter_internal_oversample

Opcje: 1 / 2 / 4  ·  Domyślnie: 1  ·  Cel: chrupanie / aliasing

Twarde limitowanie przy 44,1 kHz tworzy składowe powyżej częstotliwości Nyquista, które „zawijają się” z powrotem jako szorstkie, nieharmoniczne chrupanie. Praca limitera w 2× lub 4× usuwa większość tego aliasingu. Zalecane 2×. Koszt: większe zużycie CPU i RAM.

Jakość limitera (iteracje) (Limiter quality) → --max_iter1

Zakres: 100 … 400 (co 50)  ·  Domyślnie: 100  ·  Cel: zniekształcenia rozmycie

Limiter to iteracyjny optymalizator (FISTA), który stopniowo „dociska” sygnał do sufitu. Zbyt mała liczba iteracji = optymalizacja nie zdąży się zbiec = zostaje słyszalny błąd (chrupanie) i pre‑ringing (rozmycie transjentów). Wyższa wartość = czystszy wynik, ale dłuższe przetwarzanie. Spróbuj 200 jeśli słyszysz zniekształcenia.

Pre‑kompresja (Pre‑compression) → --pre_compression

Domyślnie: włączona  ·  Cel: pompowanie / oddychanie

Kompresor w dziedzinie głośności, działający przed limiterem. Wyrównuje najgłośniejsze fragmenty. Bywa źródłem pompowania. Odznacz, aby zachować pełną dynamikę i usunąć efekt oddychania.

Próg pre‑kompresji (dB) → --pre_compression_threshold

Zakres: 0 … 18 dB  ·  Domyślnie: 6 dB

O ile dB ponad zmierzoną głośność zaczyna działać pre‑kompresor. Wyższa wartość = kompresor reaguje rzadziej = więcej dynamiki i mniej pompowania.

Okno pre‑kompresji (s) → --pre_compression_mean_sec

Zakres: 0,05 … 1,0 s  ·  Domyślnie: 0,2 s

Długość okna uśredniania. Krótkie okno (0,2 s) potrafi „oddychać” na basie i transjentach. Wydłużenie do ~0,4 s wygładza działanie i redukuje pompowanie.
